Repository is primarily Bash-oriented at the top level with plugin/skill-driven workflow documentation. The dashboard branch uses TypeScript, Next.js App Router, Zustand store state, and a small Node websocket server. Naming is kebab-case for files, TypeScript interfaces/types for event payloads, and explicit string unions for event domains. Prefer minimal changes that preserve hard-gate workflow semantics and marker/state files under `.hive-state/`.
